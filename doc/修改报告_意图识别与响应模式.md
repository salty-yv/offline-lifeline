# 修改报告：意图识别与响应模式适配

**版本**：v0.1  
**针对阶段**：阶段 8 完成后的功能修复  
**问题分类**：Agent Runtime — PromptBuilder / ActionPlanner 输出模式硬编码

---

## 一、问题描述

### 现象

| 用户输入 | 期望表现 | 实际表现 |
|---|---|---|
| 「我迷路了，手机只有 12% 电」 | 结构化行动计划（先做这3步、当前风险…） | ✅ 正确 |
| 「怎么制造水源过滤器？」 | 分步骤的制作教程 | ❌ 仍然返回结构化求救框架 |

### 根因分析

当前 `SurvivalAgent` 的处理流程无论用户输入什么，都会经过以下固定路径：

```
用户输入
↓
RiskClassifier（若无风险关键词 → RiskDomain.UNKNOWN）
↓
ActionPlanner.buildActionStructure()（永远输出6段结构框架）
↓
PromptBuilder 把框架写入 systemInstruction 的 [Required Output Format]
↓
LLM 被强制按「先做这3步 / 当前风险 / 不要做...」格式回答
```

**关键问题在 `PromptBuilder`**：

```kotlin
// PromptBuilder.kt - 现有代码
appendLine("[Agent Action Plan]")
appendLine(actionStructure)   // ← 结构化框架被无条件注入 Prompt
```

`[Required Output Format]` 段（在 `offline_survival_app_codex_plan.md` 阶段12中定义）要求模型按固定6段回答，这在求救场景下是正确的，但对「怎么制造水源过滤器」这类**操作知识类问题**完全不适用——用户期望的是步骤化的制作指南，而非被追问电量、体力和保暖物品。

---

## 二、意图分类

用户输入可以分为两大类：

### 类型 A：求救意图（Emergency Intent）

> 用户描述的是**当前处境的危险或困难**，需要Agent给出应急行动计划。

**典型触发词**：迷路、受伤、失温、没水、地震、低电量、帮我……

**期望输出格式**：
```
先做这 3 步
当前风险
接下来几分钟
不要做
我还需要确认
什么时候必须求救
```

### 类型 B：操作意图（Procedural Intent）

> 用户询问的是**如何完成某件事**，需要Agent给出操作步骤或制作方法。

**典型触发词**：怎么…、如何…、方法、步骤、制造、制作、搭建、处理……

**期望输出格式**：
```
所需材料
制作步骤（编号列表）
注意事项
```

> [!NOTE]
> 未来还可以扩展第三类**知识查询意图**（「蛇咬了有什么症状？」），但本次修改只处理最核心的 A/B 两类。

## 三、修改方案

### 3.1 新增 `IntentClassifier`

**文件路径**：`agent/IntentClassifier.kt`

```kotlin
package com.example.offlinelifeline.agent

enum class UserIntent {
    EMERGENCY,   // 求救 / 应急处境描述
    PROCEDURAL,  // 操作步骤 / 制作方法查询
    UNKNOWN      // 无法判断，降级为 EMERGENCY 处理
}

class IntentClassifier {
    fun classify(input: String): UserIntent {
        val procedural = listOf(
            "怎么", "如何", "方法", "步骤", "制造", "制作",
            "搭建", "建造", "处理方法", "教我", "教一下",
            "怎样", "怎样才能", "能不能教", "流程", "做法"
        )
        val emergency = listOf(
            "迷路", "受伤", "出血", "帮我", "怎么办", "救命",
            "失温", "中暑", "蛇咬", "没电", "快没电", "快死了",
            "被困", "找不到路", "找不到出口"
        )

        val proceduralScore = procedural.count { input.contains(it) }
        val emergencyScore = emergency.count { input.contains(it) }

        return when {
            proceduralScore > 0 && emergencyScore == 0 -> UserIntent.PROCEDURAL
            emergencyScore > 0 -> UserIntent.EMERGENCY
            else -> UserIntent.UNKNOWN
        }
    }
}
```

> [!IMPORTANT]
> 当 `UNKNOWN` 时**降级为 EMERGENCY**，保留现有行为，确保求救场景不会漏判。

### 3.2 修改 `PromptBuilder` — 根据意图选择输出格式

**修改文件**：`agent/PromptBuilder.kt`

**新增函数 `buildOutputFormatInstruction`**，根据 `UserIntent` 返回不同的格式指令：

```kotlin
fun buildSystemInstruction(
    context: AgentContext,
    actionStructure: String,
    toolRecommendations: List<ToolRecommendation>,
    intent: UserIntent   // ← 新增参数
): String {
    return buildString {
        appendLine("你是一个完全离线运行的自救助手。")
        appendLine("你不是医生、不是救援调度系统、不能替代专业救援。")
        appendLine("你的回答必须保守、短句、步骤化、可执行。")
        appendLine("信息不足时，先给立即行动，再问最多 3 个关键问题。")
        appendLine()

        appendLine("[Known Context]")
        if (context.knownFacts.isEmpty()) {
            appendLine("暂无已确认事实。")
        } else {
            context.knownFacts.forEach { (key, value) -> appendLine("- $key: $value") }
        }
        appendLine()

        // ↓ 根据意图选择不同部分
        when (intent) {
            UserIntent.EMERGENCY, UserIntent.UNKNOWN -> {
                appendLine("[Agent Action Plan]")
                appendLine(actionStructure)
                appendLine()
                appendLine("[Available Local Tools]")
                if (toolRecommendations.isEmpty()) {
                    appendLine("暂无推荐工具。")
                } else {
                    toolRecommendations.forEach {
                        appendLine("- ${it.toolType.name}: ${it.reason}")
                    }
                }
                appendLine()
                appendLine("[Required Output Format]")
                appendLine("请严格按以下结构回答：")
                appendLine("1. 先做这 3 步")
                appendLine("2. 当前风险")
                appendLine("3. 接下来几分钟")
                appendLine("4. 不要做")
                appendLine("5. 我还需要确认")
                appendLine("6. 可使用的本地工具")
            }

            UserIntent.PROCEDURAL -> {
                // 操作类问题：不注入应急框架，只注入步骤格式要求
                appendLine("[Required Output Format]")
                appendLine("用户在询问操作方法或制作步骤，请直接回答，不要套用应急框架。")
                appendLine("请按以下格式回答：")
                appendLine("1. 所需材料（如有）")
                appendLine("2. 步骤（用编号列出，每步一行，简洁）")
                appendLine("3. 注意事项（如有，不超过 3 条）")
                appendLine("不要追问用户的电量、体力或装备，除非问题与处境直接相关。")
            }
        }
    }
}
```

### 3.3 修改 `SurvivalAgent` — 传入意图

**修改文件**：`agent/SurvivalAgent.kt`

```kotlin
class SurvivalAgent(
    private val contextManager: ContextManager,
    private val riskClassifier: RiskClassifier,
    private val intentClassifier: IntentClassifier,   // ← 新增
    private val questionPlanner: QuestionPlanner,
    private val actionPlanner: ActionPlanner,
    private val toolRouter: ToolRouter,
    private val promptBuilder: PromptBuilder,
    private val safetyKernel: SafetyKernel,
    private val llmEngine: LocalLlmEngine
) {
    fun prepareResponse(userInput: String, history: List<ChatMessage>): PreparedAgentResponse {
        val scopedHistory = contextManager.summarizeIfNeeded(history)
        val contextMessages = scopedHistory + ChatMessage(
            id = "current-user-input",
            role = ChatRole.USER,
            text = userInput,
            createdAtMillis = 0L
        )
        val baseContext = contextManager.buildContext(contextMessages)
        val risks = riskClassifier.classify(userInput, baseContext)
        val context = baseContext.copy(riskDomains = risks)

        val intent = intentClassifier.classify(userInput)   // ← 新增

        val questions = questionPlanner.planQuestions(context)
        val tools = toolRouter.recommendTools(risks, context)
        val actionStructure = actionPlanner.buildActionStructure(risks, context, questions)

        val systemInstruction = promptBuilder.buildSystemInstruction(
            context, actionStructure, tools,
            intent = intent   // ← 新增
        )
        val safetyInstruction = promptBuilder.buildSafetyInstruction(
            safetyKernel.buildSafetyInstruction(risks)
        )

        return PreparedAgentResponse(
            request = InferenceRequest(
                text = userInput,
                systemInstruction = systemInstruction,
                safetyInstruction = safetyInstruction
            ),
            response = AgentResponse(
                text = actionStructure,
                riskDomains = risks,
                followUpQuestions = questions,
                toolRecommendations = tools
            )
        )
    }
    // sendMessage 不需要修改
}
```

**输入**：`我受伤了，怎么止血？`

## 六、设计决策说明

### 为什么不用 LLM 自己判断意图？

在用 Prompt 注入格式指令之前让 LLM 先做意图判断，需要额外一次推理调用，增加延迟和电量消耗，与本项目「紧急场景、低电量优先」的定位相悖。规则分类器更轻量、更确定。

### 为什么 UNKNOWN 降级为 EMERGENCY？

应急类工具宁可对普通问题过于谨慎，也不能对真正求救的用户失效。

### 为什么不改 ActionPlanner？

`ActionPlanner.buildActionStructure()` 的输出对 EMERGENCY 场景仍然有用（作为 `[Agent Action Plan]` 注入 Prompt）。只需要在 PROCEDURAL 时让 `PromptBuilder` 不把它注入格式要求部分即可，无需修改 ActionPlanner 本身。

## 七、后续扩展建议（非本次范围）

1. **知识查询意图**（`KNOWLEDGE`）：「蛇咬了有什么症状？」→ 输出简洁知识卡片格式
2. **混合意图**：「我在野外迷路了，怎么找水？」→ 优先 EMERGENCY，但 Prompt 里补充「同时给出找水方法」
3. **多轮意图跟踪**：如果上轮是 PROCEDURAL，下轮用户说「好的」，延续上轮意图而非重新分类
